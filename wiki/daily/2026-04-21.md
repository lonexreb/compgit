# 2026-04-21

## Phase 1 shipped — Chrome extension MVP

All seven implementation steps green:

1. Shared TypeScript core — GitHub client, cache, auth, aggregation, time. 41 Vitest tests, ~97% coverage.
2. WXT scaffold — React + Tailwind v4 + self-hosted Fraunces + JetBrains Mono. Build passes.
3. Options page — debounced live PAT validation, auto-save, clear-cache / refresh-now buttons.
4. Background worker — `chrome.alarms` every 15 min, typed message protocol, `refreshContributions()` DI-able.
5. Popup — today's Fraunces numeral, 7-day sparkline, streak line, offline badge, error fallback.
6. Side panel — Today / Heatmap / Trends tabs with keyboard-accessible navigation and URL-hash state.
7. Error states — "couldn't sync" surface in both popup and side panel when the background fetch fails.

## Phase 2 started — Swift core shipped

[[../entities/Swift-Core]] is live. Mirrors [[../entities/Shared-TypeScript-Core]] structurally:

- `GitHubClient` (actor) — GraphQL over URLSession, typed `CompgitError`.
- `SharedStore` — App-Group `UserDefaults` with the same storage keys as Chrome.
- `KeychainPAT` — Keychain Services wrapper with optional accessGroup for widget sharing.
- `Aggregate` + `Time` — pure ports of the TS aggregation and date helpers.
- `Errors` — single `CompgitError` enum covering auth / rateLimited / network / validation / notAuthenticated.

`swift build` is clean on Command Line Tools 16. The surface area the iOS/macOS apps + widgets will consume is ready.
