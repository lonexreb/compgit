# apps/ios

Phase 2b iOS host app + widget extension. The full architecture lives in [`../../wiki/entities/Apple-Surfaces.md`](../../wiki/entities/Apple-Surfaces.md).

This directory currently holds the project specification only ‚Äî the `.xcodeproj` and Swift source files arrive when full Xcode is installed (see [`../../NEXT-TO-DO.md`](../../NEXT-TO-DO.md) ‚Üí Phase 2b).

## Generate the Xcode project

```bash
brew install xcodegen   # one-time
cd apps/ios
xcodegen generate
open Compgit.xcodeproj
```

The `.xcodeproj/` is generated and gitignored. The source of truth is [`project.yml`](./project.yml).

## Before you generate

Replace `TEAMID` in `project.yml` with the real Apple Developer Team ID. Two places:

1. `settings.base.DEVELOPMENT_TEAM`
2. `APP_GROUP_ID` and the entitlements `com.apple.security.application-groups`

Renaming after the fact cascades through every Apple Developer entitlement ‚Äî pick the Team ID once, before you run `xcodegen` for the first time.

## Targets

| Target | Bundle ID | Role |
|---|---|---|
| `CompgitApp` | `com.lonexreb.compgit` | iOS 17+ host app ‚Äî PAT entry, manual refresh, last-synced timestamp. Registers `BGAppRefreshTask` for the 30-min refresh loop. |
| `CompgitWidgetExtension` | `com.lonexreb.compgit.widget` | iOS Home Screen / Lock Screen widget. Reads from `SharedStore(suiteName: "group.TEAMID.compgit")`; never networks. |

Both targets depend on the local SPM package `packages/shared-swift` ‚Äî `CompgitCore` for runtime + `CompgitSchema` for generated models.

## Entitlements

- **App Groups** ‚Äî `group.TEAMID.compgit`. Backs `SharedStore`'s suite.
- **Keychain Sharing** ‚Äî `$(AppIdentifierPrefix)compgit`. Lets the widget read the PAT the host saved via `KeychainPAT(accessGroup:)`.
- **BackgroundModes ‚Üí fetch** ‚Äî host target only. Required for `BGAppRefreshTask`.

The plist + entitlements files referenced from `project.yml` are written when Phase 2b lands ‚Äî `xcodegen` will create the empty placeholders, then 2b.3 / 2b.4 fill them.
