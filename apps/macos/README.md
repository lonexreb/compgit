# apps/macos

Phase 2b macOS host app + desktop widget extension. The full architecture lives in [`../../wiki/entities/Apple-Surfaces.md`](../../wiki/entities/Apple-Surfaces.md).

This directory currently holds the project specification only ‚Äî the `.xcodeproj` and Swift source files arrive when full Xcode is installed (see [`../../NEXT-TO-DO.md`](../../NEXT-TO-DO.md) ‚Üí Phase 2b).

## Generate the Xcode project

```bash
brew install xcodegen   # one-time
cd apps/macos
xcodegen generate
open CompgitMac.xcodeproj
```

The `.xcodeproj/` is generated and gitignored. The source of truth is [`project.yml`](./project.yml).

## Before you generate

Replace `TEAMID` in `project.yml` with the real Apple Developer Team ID ‚Äî the same ID used in [`../ios/project.yml`](../ios/project.yml) so both surfaces share the App Group `group.TEAMID.compgit`.

## Targets

| Target | Bundle ID | Role |
|---|---|---|
| `CompgitMacApp` | `com.lonexreb.compgit.mac` | macOS 14+ host app ‚Äî same shape as iOS, plus a `MenuBarExtra` stub for Phase 5. App-sandboxed with `network.client` enabled (only the host fetches; widget never networks). |
| `CompgitMacWidgetExtension` | `com.lonexreb.compgit.mac.widget` | Desktop widget. App-sandboxed; reads from `SharedStore(suiteName: "group.TEAMID.compgit")` only. |

Both targets depend on the local SPM package `packages/shared-swift` (`CompgitCore` + `CompgitSchema`).

## Entitlements

- **App Sandbox** ‚Äî required for Mac App Store; both targets enable it.
- **Network Client** ‚Äî host target only. Widget target has it explicitly disabled to enforce the "widgets never network" invariant from [Apple-Surfaces](../../wiki/entities/Apple-Surfaces.md).
- **App Groups** ‚Äî `group.TEAMID.compgit`. Same ID as iOS so a future cross-device sync layer can share storage shape.
- **Keychain Sharing** ‚Äî `$(AppIdentifierPrefix)compgit`. Same access group as iOS.
- **Hardened Runtime** ‚Äî required for Developer ID-signed direct distribution (the notarized `.dmg` path in [Distribution-And-Release](../../wiki/entities/Distribution-And-Release.md)).
